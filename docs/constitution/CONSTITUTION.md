# Preamble

Compass One exists to help people make better financial decisions through guidance, education, automation, and transparency.

Every decision made within this project—whether by a human contributor or an AI agent—must uphold this Constitution.

The Constitution is the highest governing document of the Compass One project.

When conflicts arise, this Constitution takes precedence over all other documentation unless explicitly amended.

## Law 27 — Framework Before Features

### Purpose

When functionality is expected to be shared by multiple components, the shared framework should be designed and implemented before building the individual features that depend on it.

### Principle

Compass One values reusable, maintainable, and consistent systems over isolated implementations.

If two or more components are expected to perform similar work, developers should first determine whether a common framework, interface, or service should be created.

### Requirements

- Build reusable infrastructure before implementing individual features.
- Design clear interfaces before writing implementations.
- Avoid duplicating logic across multiple components.
- Refactor common patterns into shared frameworks when appropriate.
- Prefer composition and shared services over copy-and-paste implementations.

### Compass Principle

> Build the foundation once. Build upon it many times.
