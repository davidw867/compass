Contributing to Compass One

Thank you for your interest in contributing.

Compass One is built around long-term maintainability, security, and responsible financial software. Every contribution should improve the project without compromising these principles.

Before You Start

Please:

- Read the project documentation.
- Search existing issues before opening a new one.
- Discuss major architectural changes before implementing them.

Coding Standards

- Follow Clean Architecture principles.
- Keep components modular.
- Write readable, maintainable code.
- Add or update tests whenever appropriate.
- Document public APIs.

Pull Requests

Each pull request should:

- Solve one problem.
- Include clear commit messages.
- Pass all automated tests.
- Include documentation updates when necessary.

Security

Never commit:

- API keys
- Secrets
- Passwords
- Personal information
- Production credentials

AI Contributions

AI-assisted code is welcome, but contributors are responsible for reviewing, testing, and understanding all generated code before submission.

Philosophy

We value:

- Security over convenience
- Simplicity over complexity
- Transparency over magic
- Long-term maintainability over short-term speed
- Responsible AI development
