Security Policy

Supported Versions

The latest stable release of Compass One will receive security updates.

Development branches may change rapidly and should not be considered production-ready.

Reporting a Vulnerability

Please do not publicly disclose security vulnerabilities before they have been reviewed.

Instead:

1. Contact the maintainers privately.
2. Include detailed reproduction steps.
3. Include affected versions if known.
4. Include proof-of-concept code only when necessary.

We will:

- Acknowledge reports promptly.
- Investigate responsibly.
- Provide updates during the review process.
- Credit responsible disclosure when appropriate.

Security Principles

Compass One follows a security-first philosophy.

Key principles include:

- Least privilege
- Secure defaults
- Defense in depth
- Encryption of sensitive data
- Auditability
- Responsible AI governance

Security is considered a core feature rather than an optional enhancement.
